## 现状与原因
- 相机初始位置与目标：`src\components\ParticleScene.tsx:689` 设置位置，`668` 设置目标为 `(0,0,0)`。
- 若仍感觉“有点斜”，通常是由于上次交互留下的旋转（OrbitControls）或用户手势造成的初始角度偏移；也可能希望严格锁定到正面而不允许初始偏转。

## 方案 A（保持可旋转，确保初始正面）
- 使用已存在的 `initialCamera` 配置，明确设定正面：
  - 在 `src\App.tsx:28` 设置 `initialCamera = { radius: 80, theta: 0, phi: 0 }`，对应相机位于 `z=80`、对准原点。
- 初次加载时强制复位：
  - 已有 `resetViewTick` 触发逻辑（`src\components\ParticleScene.tsx:674-687`）；在进入页面或需要时点击“重置视角”（`src\App.tsx:122`）即可回到正面。

## 方案 B（新增“锁定正面”开关）
- 新增 `frontLock?: boolean`，开启后：
  - `OrbitControls` 设置 `enableRotate={false}`（或限制角度：`minPolarAngle===maxPolarAngle`、`minAzimuthAngle===maxAzimuthAngle`）以禁止初始旋转，始终保持正面。
  - 初始相机位置仍由 `initialCamera` 决定（默认正面）。
- UI 增加一个“锁定正面”开关以便随时启用/关闭。

## 交付内容
- A 方案无需新增文件，仅确认 `initialCamera` 为正面并说明使用“重置视角”。
- B 方案将：
  - 更新 `ParticleScene` 支持 `frontLock`，限制 `OrbitControls` 旋转。
  - 在工具栏或调参面板增加对应开关。
