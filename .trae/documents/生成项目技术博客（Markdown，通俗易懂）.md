## 结构大纲
1. 引言：什么是粒子系统（类比星空、烟雾），我们为什么需要它
2. 能力一览：四种模式（随机/文本/图片/GLTF）与多种特效（爆裂、涡旋、冲击波、黑洞、瀑布、彩虹、风场、呼吸、波纹、球壳、网格）
3. 技术栈选择：React + TypeScript + Vite + three + @react-three/fiber + @react-three/drei + 后期 Bloom
4. 架构总览：入口与页面、核心场景组件、图片采样工作线程、上传面板与参数面板、资源与预设（仅文字概述，不列路径）
5. 渲染管线：Canvas/R3F、粒子几何（Points/BufferGeometry/颜色属性）、点精灵着色器（圆形、遮罩空洞、彩虹色相、设备像素比）
6. 图片如何变成粒子：采样流程与关键参数（Gamma、亮度阈值分位、饱和度阈值、透明度、白背景剔除）；为何使用工作线程提升流畅度
7. 文本与模型采样：文字转像素再转粒子，GLTF 顶点稀疏采样与统一缩放的直观解释
8. 特效的原理：
   - 爆裂回流：由中心向外的速度脉冲与回归吸引
   - 星环涡旋：围绕中心的切向速度场（类比旋风）
   - 光爆冲击波：随半径传播的正弦波动（类比声波环）
   - 黑洞吞噬：向中心收敛，遮罩形成可见“空洞”
   - 瀑布云幕/风洞回旋/星群聚散/波纹干涉/球壳/网格阵列：逐一用通俗语言解释速度更新逻辑
9. 交互与参数：工具栏切换、调参项（粒子尺寸、Bloom 阈值/强度、反中心斥力、遮罩半径与软硬、运动速度、相机角度、粒子数量）、截图与视角重置、FPS 显示
10. 性能与优化建议：
    - 粒子数量与尺寸的权衡；damping 与速度步长
    - Bloom 对性能的影响与适度使用
    - CPU/Worker 与 GPU/着色器的分工
    - 资源大小与网络开销的注意点
11. 快速上手与扩展：
    - 安装与运行命令（`npm install`、`npm run dev`、`npm run build`、`npm run preview`）
    - 如何新增一个特效：给出伪代码框架与思路
    - 如何接入音频或新增参数面板项
12. 常见问题：
    - 图片过暗/过亮如何调：Gamma 与阈值
    - 白底图像如何去白：跳过低饱和高亮背景
    - 卡顿如何优化：减少粒子、调小点尺寸、关闭/降低 Bloom
    - 端口占用与预览方式
13. 总结与下一步：面向更复杂的展陈场景（多图序列、镜头关键帧、热点卡片等）的可能扩展
